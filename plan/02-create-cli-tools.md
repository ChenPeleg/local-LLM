
# LM Studio CLI tools

This file lists the primary `lms` CLI commands referenced from https://lmstudio.ai/docs/cli

- `lms chat` — Start an interactive chat with a model
- `lms get` — Search for and download models
- `lms load` — Load a model (options: `--gpu`, `--context-length`, `--identifier`)
- `lms unload` — Unload a model (option: `--all`)
- `lms ls` — List models available on disk
- `lms ps` — List models currently loaded in memory
- `lms import` — Import a model file into LM Studio

- `lms server` — Server management commands (subcommands: `start`, `stop`, `status`)
- `lms log` — View/log incoming and outgoing messages (`log stream`)

- `lms runtime` — Manage and update the inference runtime

- `lms clone` — Clone an artifact from LM Studio Hub to a local folder
- `lms push` — Upload the artifact in the current folder to LM Studio Hub
- `lms dev` — Start a plugin development server (dev tools)
- `lms login` — Authenticate with LM Studio (login / auth)

For full details and subcommands see the official docs: https://lmstudio.ai/docs/cli
